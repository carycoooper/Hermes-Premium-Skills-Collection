## 🚀 快速开始

### 前置要求

- ✅ **Hermes Agent** 已安装 ([安装指南](https://hermes-agent.nousresearch.com/docs/installation))
- ✅ **Python 3.8+** 或 **Node.js 18+**
- ✅ **Git** (用于克隆仓库)

### 一键安装（推荐）

```bash
# 克隆仓库到本地
git clone https://github.com/YOUR_USERNAME/hermes-skills.git
cd hermes-skills

# 复制所有 skills 到 Hermes 目录
cp -r skills/* ~/.hermes/skills/

# 重启 Hermes Agent
hermes restart

# 验证安装
hermes chat --toolsets skills -q "What skills are available?"
```

### 手动安装单个 Skill

```bash
# 例如：只安装 Deep Research
mkdir -p ~/.hermes/skills/research/
cp -r skills/research/deep-research ~/.hermes/skills/research/

# 或者：从 Releases 下载单个 skill 文件
```

### Docker 安装

```dockerfile
FROM nousresearch/hermes-agent:latest

# 复制 skills
COPY skills/ /root/.hermes/skills/

# 设置权限
RUN chmod -R 755 /root/.hermes/skills/
```

<details>
<summary><b>Hermes Skill 架构详解（点击展开）</b></summary>

#### 什么是 Hermes Skill？

Hermes Skill 是一种**渐进式披露（Progressive Disclosure）**的知识模块：

```
Level 0: 元数据层 (~3000 tokens)
├── 技能名称、描述、分类
├── 触发条件
└── 权限需求

Level 1: 完整指令 (按需加载)
├── SKILL.md 全文
├── 流程步骤
├── 模板和示例
└── 最佳实践

Level 2: 参考文件 (深层需求)
├── API 文档
├── 代码示例
└── 外部资源链接
```

#### SKILL.md 文件结构

```yaml
---
name: skill-name                    # 技能标识符
description: Brief description      # 功能描述 (<1024字符)
version: 1.0.0                      # 版本号
license: MIT                        # 开源协议
platforms: [macos, linux]           # 平台限制（可选）
metadata:
  hermes:
    tags: [category, keywords]      # 标签用于搜索
    related_skills: [other-skill]   # 关联技能
    requires_toolsets: [web]        # 依赖的工具集
    config:                          # 可配置项
      - key: setting
        default: value
---
# Markdown 内容（完整指令）
```

## 🔧 配置与定制

### 全局配置

创建 `~/.hermes/config/skills-config.yaml`:

```yaml
# 全局设置
defaults:
  language: auto              # auto / en / zh
  privacy_mode: enhanced      # standard / enhanced
  telemetry: false            # 是否匿名上报使用统计

# 各技能默认配置
skill_defaults:
  deep-research:
    max_sources: 10
    depth_level: standard
  
  mission-control:
    briefing_time: "07:00"
    include_weather: true
  
  email-management:
    triage_categories: urgent,important,to-read,newsletter
    
  github-integration:
    pr_review_depth: standard
    
  finance-tracker:
    currency: USD
    privacy_mode: true
```

### 单个 Skill 配置

每个 Skill 都支持独立的配置文件：

```yaml
# ~/.hermes/skills/productivity/self-improving-agent/config.yaml
promotion_threshold: 3
memory_retention_days: 30
auto_export_weekly: true
max_permanent_memories: 100
```

### 环境变量配置

部分 Skill 需要外部 API 密钥：

```bash
# GitHub Integration
export GITHUB_TOKEN="ghp_xxxxxxxxxxxx"

# Email Management (Gmail)
export EMAIL_ADDRESS="you@gmail.com"
export EMAIL_APP_PASSWORD="xxxx"

# Finance Tracker (可选，用于在线银行API)
# 本地CSV处理不需要
```

**安全建议：**
- 使用环境变量或密钥管理器（如 1Password CLI）
- 不要将密钥提交到 Git 仓库
- 定期轮换 API 密钥

### 相比 OpenClaw 原版的 20 项重大升级

#### 🛡️ 1. 安全性增强

| 特性 | OpenClaw | Hermes Edition |
|------|----------|----------------|
| 权限粒度 | 基础读写 | 分层（读取/草稿/发送/删除） |
| 数据隐私 | 未明确 | 本地处理+加密+自动清理 |
| 回滚机制 | 无 | 一键回滚任意版本 |
| 审计日志 | 基础 | 完整操作轨迹+时间戳 |
| 人机协同 | 低 | 高风险操作必须人工确认 |

#### 🧠 2. 智能化提升

- **Self-Improving Agent**: 新增冲突检测、记忆衰减、隐私标记
- **Capability Evolver**: 增加 A/B 测试模拟、影响度量、进化疲劳防护
- **Deep Research**: 引入证据权重、矛盾矩阵、置信度评级
- **Email Management**: 三维分类替代二维、智能草稿多语气

#### 🎨 3. 用户体验优化

- **统一模板系统**: 每个技能提供 3-5 种输出格式
- **渐进式 disclosure**: TLDR → 标准 → 详细 → 原文
- **可视化辅助**: ASCII 图表、表格、进度指示器
- **错误恢复**: 清晰的错误消息+解决方案+回退路径

#### 🔌 4. 集成生态

- **Skill 间协作**: 明确定义数据流和集成点
- **Toolset 感知**: 条件激活（`requires_toolsets`）
- **外部服务**: 标准化的 API 集成模式
- **插件兼容**: 支持命名空间避免冲突

### 🆕 创新功能（OpenClaw 没有）

1. **📊 能力热力图** (`capability-evolver`)
   - 可视化展示哪些能力在进化
   - 识别能力盲区

2. **🧠 记忆健康度仪表盘** (`self-improving-agent`)
   - 监控记忆质量、新鲜度、利用率
   - 自动清理过期/低价值记忆

3. **🔍 研究可信度评分** (`deep-research`)
   - 为每个信息源打分（权威性/时效性/偏见）
   - 综合研究报告的可信度等级

4. **📧 邮件情感分析** (`email-management`)
   - 检测邮件情绪（紧急/友好/不满）
   - 建议最佳回复时机和语气

5. **🐙 PR 影响范围分析** (`github-integration`)
   - 自动识别受影响的代码区域
   - 评估回归风险

6. **🎨 设计系统一致性检查** (`frontend-design`)
   - 对比组件与设计令牌的差异
   - 确保 Design System 合规性

7. **💰 财务异常根因分析** (`finance-tracker`)
   - 不仅标记异常，还解释可能原因
   - 关联生活事件（旅行、节假日等）

8. **🛠️ 批量操作沙盒** (`utility-toolkit`)
   - 所有破坏性操作先干运行
   - 可视化预览变更

## 🤝 贡献指南

我们欢迎社区贡献！无论是新技能、Bug修复还是文档改进。

### 如何贡献新 Skill？

1. **Fork 本仓库**
2. **创建分支**: `git checkout -b feature/amazing-new-skill`
3. **遵循规范**:
   - 使用标准的 SKILL.md 格式
   - 包含完整的 YAML frontmatter
   - 提供 Quick Reference 表格
   - 编写 Procedure（分步骤流程）
   - 添加 Pitfalls & Best Practices
   - 包含 Verification Checklist
4. **添加测试用例**（如果适用）
5. **提交 PR**: 详细描述你的 Skill 的价值和用途

### Skill 质量标准

我们遵循严格的质量门槛：

- ✅ **完整性**: 覆盖主要使用场景（80/20 法则）
- ✅ **清晰性**: 初学者能在 5 分钟内上手
- ✅ **安全性**: 不泄露敏感数据，权限最小化
- ✅ **可维护性**: 代码/指令易于理解和修改
- ✅ **可测试性**: 提供验证清单
- ✅ **文档化**: 注释充分，示例丰富

### 代码规范

- 使用 **Markdown** 编写 SKILL.md
- **YAML frontmatter** 必须完整
- **英文为主**（Skill 名称和元数据），**内容可双语**
- 遵循 **Hermes 官方 Skill 规范**
